## 第3页 | 项目背景与问题定义

### AI Skill 安全挑战

随着 AI Agent 与 Skill 生态的快速发展，Skill 已成为大模型能力扩展的核心载体。然而，Skill 的开放分发机制也带来了前所未有的安全风险：

| 风险维度 | 典型场景 | 危害等级 |
|---------|---------|---------|
| 代码层威胁 | 恶意域名通信、数据外传、命令注入、硬编码密钥 | 🔴 高 |
| 语义层威胁 | 提示注入、心理引导、功能不符、越权访问 | 🔴 高 |
| 供应链威胁 | 依赖包篡改、字节码篡改、触发器劫持 | 🟠 中高 |
| 隐蔽攻击 | Unicode 隐写、代码混淆、攻击链组合 | 🔴 高 |

### 现有方案的不足

- **纯静态扫描**：无法检测语义层威胁（提示注入、心理引导等）
- **纯 LLM 分析**：成本高、速度慢，且存在误报
- **缺乏关联分析**：单点检测无法发现跨文件攻击链
- **误报率高**：正则匹配缺乏上下文理解，大量误报干扰判断

## 第4页 | 项目概述

### SkillGuard 是什么

SkillGuard 是一款**基于多维度安全分析技术的 AI Skills 安全检测平台**，为 Skill 用户、开发者和安全研究人员提供专业的安全验证与扫描服务。

### 核心设计理念

```
L1 静态候选  →  L2 LLM 静态审查  →  L3 LLM 关联分析
   (广覆盖)        (精准判别)          (深度关联)
```

**三层递进式检测架构**，结合 **ThreatIntelAgent 威胁情报研判**，实现从代码层到语义层的全方位安全覆盖。

### 关键数据

| 指标 | 数值 |
|------|------|
| 威胁检测类型 | **22 类**（代码层 12 + 语义层 10） |
| 内置检测规则 | **16 条** |
| 内置恶意域名 | **6 个** |
| 检测方法标识 | **9 种** |
| 风险定级 | **5 级**（安全/警告/危险/高危/未知） |
| 报告格式 | **3 种**（MD/JSON/HTML） |

## 第7页 | 三层递进式检测引擎（核心创新）

### 检测流程

```
                    Skill 文件上传
                         │
                    ┌────▼─────┐
                    │  L1 广覆盖 │  正则规则扫描 16 条
                    │  静态候选  │  恶意域名匹配 6 个
                    │           │  → 生成候选片段
                    └────┬─────┘
                         │
                    ┌────▼─────┐
                    │  L2 精准判别│  逐一 LLM 审查
                    │  LLM审查  │  判定真阳性/误报
                    │           │  调整严重程度
                    └────┬─────┘
                         │
                    ┌────▼─────┐
                    │  L3 深度关联│  跨文件关联分析
                    │  LLM关联  │  攻击链识别
                    │           │  供应链模式检测
                    └────┬─────┘
                         │
                    ┌────▼─────┐
                    │  结果输出  │  去重合并
                    │           │  风险评分
                    │           │  报告生成
                    └──────────┘
```

### 并行检测维度

| 检测维度 | 方法 | 检测目标 |
|---------|------|---------|
| 静态代码分析 | 正则规则匹配 | 代码层 12 类威胁 |
| 威胁特征匹配 | IOC 域名库 + Agent 研判 | 恶意域名真实威胁判定 |
| LLM 意图分析 | 千问大模型语义理解 | 语义层 10 类威胁 |
| 沙箱行为分析 | RestrictedPython 隔离执行 | 受限操作与异常行为 |

### ThreatIntelAgent 研判机制

域名命中后不是简单告警，而是调用 LLM 分析上下文：

```
域名命中 → 提取上下文 → LLM研判 → 判定结果
                         │
              ┌──────────┼──────────┐
              │          │          │
          真实威胁    无害引用    调整严重度
          (告警)     (标记误报)   (升降级)
```

## 第9页 | 技术亮点与创新

### 亮点一：三层递进式检测架构

**问题**：传统方案要么广覆盖高误报（纯正则），要么高精度低覆盖（纯 LLM）

**创新**：
- L1 正则广覆盖收集候选 → L2 LLM 逐一审查去误报 → L3 LLM 跨文件关联
- 每层递进，精度逐步提升，兼顾**覆盖率**与**准确率**

### 亮点二：ThreatIntelAgent 智能研判

**问题**：域名命中不等于真实威胁（可能是注释、文档示例、测试代码）

**创新**：
- 域名命中后自动提取上下文，调用 LLM 分析真实威胁程度
- 区分 C2 通信、数据外传、钓鱼重定向等攻击意图
- 自动调整严重程度，显著降低误报率

### 亮点三：LLM 降级容错机制

**问题**：LLM API 可能超时、限流、服务不可用

**创新**：
- 指数退避重试（2^n 秒间隔，最多 2 次）
- 全部失败时标记为 `unknown`，明确告知用户结果不可靠
- 部分失败时记录降级日志，提示结果仅供参考
- 静态分析结果始终可用，不会因 LLM 故障而完全失效

### 亮点四：智能内容分块

**问题**：大文件超出 LLM Token 限制

**创新**：
- 按函数/类边界智能分块，保留语义完整性
- 优先按 `def`/`class`/`function` 顶格行分割
- 单块超限时再按字符硬切
- 无代码结构时按段落/空行分割

### 亮点五：发现去重合并

**问题**：同一威胁被多种方法重复检测

**创新**：
- 同一文件 + 同一威胁类型 + 行号相近（差 ≤ 3）→ 自动合并
- 合并后标注多个检测方法（如 `static_analysis+llm_analysis`）
- 取最高严重程度，保留最精确行号

### 亮点六：LLM 个性化安全建议

**问题**：传统报告只有通用模板建议

**创新**：
- 扫描完成后调用 LLM 根据具体威胁生成 3-5 条针对性建议
- 每条建议包含问题描述、修复方案、优先级
- LLM 失败时降级到模板建议，确保报告完整性

## 第10页 | 安全防护机制

### 平台自身安全

| 安全机制 | 防护目标 | 实现方式 |
|---------|---------|---------|
| Zip Slip 防护 | 压缩包路径穿越攻击 | 解压时校验每个成员的绝对路径 |
| Tar Slip 防护 | TAR 包路径穿越 + 符号链接 | 校验路径 + 跳过符号链接 |
| 文件名安全处理 | 路径注入 | `secure_filename()` 清洗 |
| 上传大小限制 | DoS 攻击 | 50MB 上传限制 |
| 僵尸任务清理 | 资源泄露 | 启动时清理未完成任务 |
| 内容哈希缓存 | 重复扫描消耗 | SHA-256 哈希比对，命中即返回 |

### 检测可靠性保障

```
LLM 调用链路：
  调用 → 超时重试(指数退避) → 降级标记 → 日志记录

扫描结果可信度：
  LLM 全部成功 → 正常结果
  LLM 部分失败 → 标记"部分降级"，结果仅供参考
  LLM 全部失败 → 标记"unknown"，明确告知不可靠
```
